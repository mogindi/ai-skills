# Production Security Audit

Audit production services with a bias toward evidence and operational risk. Keep the output short enough for an approval workflow while preserving the concrete findings needed for remediation.

## Workflow

1. Identify the service, environment, ownership, data classification, and release or approval context.
2. Review identity, network exposure, secret handling, logging, backup, runtime hardening, and incident-readiness evidence.
3. Distinguish blocking production risks from recommended hardening work.
4. Record validation evidence and gaps, including commands or artifacts reviewed.
5. Require security ownership review for high-risk models, sensitive data, or externally exposed production changes.
